# BOJ Workflow (C++ / Python)

대상: 백준 기반 코딩테스트 준비를 저장소에 지속적으로 누적하는 워크플로우

## 1. 문제 선택

| 순서 | 작업 | 체크포인트 |
|---|---|---|
| 1 | 주간 목표(예: 10문제) 설정 | 난이도는 `브론즈 -> 실버` 점진 상승 |
| 2 | 유형 1개 고정(예: 문자열, DP) | 같은 유형을 연속 3~5문제로 묶기 |
| 3 | 언어 선택 | 1차 C++, 2차 Python(또는 반대) |

## 2. 풀이 사이클(문제 1개 기준)

| 단계 | 작업 | 시간 가이드 |
|---|---|---|
| A | 문제 이해/입출력 정리 | 5~10분 |
| B | 손풀이/반례 도출 | 10분 |
| C | 1차 구현 | 20~30분 |
| D | 로컬 테스트(기본/반례) | 10분 |
| E | 제출/오답 분석 | 10분 |
| F | 재구현(다른 언어) | 20~30분 |
| G | 핵심 회고 3줄 기록 | 3분 |

## 3. 저장소 반영 규칙(권장)

| 항목 | 규칙 |
|---|---|
| 디렉터리명 | `topic/문제번호_YYMMDD_이름/` |
| 코드 파일 | `main.cpp`, `main.py` 또는 문제명 명시 파일 |
| 메모 | `notes.md`에 실수/반례/복잡도 기록 |
| 커밋 | 문제 단위 1커밋 |

## 4. 주간 운영 순서표

| 요일 | 운영 |
|---|---|
| 월~목 | 신규 문제 풀이(유형 집중) |
| 금 | 틀린 문제 재풀이 |
| 토 | 같은 문제 언어 전환 풀이(C++ <-> Python) |
| 일 | 주간 회고 + 다음 주 유형 선정 |

## 5. 품질 체크리스트

- 시간복잡도/공간복잡도 한 줄 설명이 가능한가
- 최소 반례 3개를 직접 만들었는가
- 같은 문제를 다른 언어로 다시 풀어봤는가
- 1주 뒤 다시 봐도 이해되는 코드 구조인가
